我今天开始运营的这个网站，源文件继承自过去的博客“朝闻道”。我决定停止对“朝闻道”的维护，所以域名 zhaowendao.art 直接就拿来为这个网站所用了，好消息是这个域名年底才会过期（按原计划一年一换，省钱）。

我今天在 Obsidian 下载了 Thino 的插件，它能承担 flomo 的功能，所以我再次弃用并卸载了 flomo，这次应该不会再吃回头草了。

这个网站能帮我做什么呢？

  • 写日记。虽然前不久我补了6块差价，买了一本日记的 webdav 同步，但是时至今日，我一篇日记都没写出来。我希望自己能爱上在博客用键盘打字写日记的感觉，如果一切顺利（每天在这儿写点什么），那么我就可以专门在“一本日记”APP里记录反思和隐私内容。
  • 当做社交平台来用。我一直在纠结用微博还是用推特、发小红书还是买可话会员。想找一个地方随心所欲写点东西，且能相信平台会愿意将我的数据交给我。不想看平台的广告、也不想被平台推荐各种信息垃圾。
  • 写点碎碎念。哪怕是只写一句话，也能解开我这个心病——很多时候，我都想在网上写点什么，让别人看见，但又不想他们真的看见。我也很久很久没有写过碎碎念了，无论是社交平台还是笔记软件，倾诉的需求得不到释放，每天都感到憋屈。

22:09 我刚刚灵机一动，想到了用 Github + jsdelivr 作图床的方法，经过测试方法可行。方法很简单，将图片上传到一个 Github 仓库，然后在浏览器地址栏输入下面的网络地址，就可以将该仓库的文件上传到 jsdelivr 的服务器，获取该文件在 jsdelivr 服务器上的网络地址，就可以在网站随意引用！

https://cdn.jsdelivr.net/gh/你的Github用户名/你的Github仓库名/

用这个网站做点有意思的事儿，我可以：

  • 每天分享一首歌；
  • 每天分享一张图；
  • 每天分享一个高级词汇；
  • 每天分享一个成语；
  • 每天分享一句名言；

前两个真是分享，后三个说是学习更合适，打着“分享”的旗号督促自己去学习或整理。

我对个人管理的规划是：

  • 在 Github 上日更，既当做社交平台、又当做日记本、还当做碎碎念。
  • 用 iBetter 管理和提醒自己坚持好习惯。
  • 在小红书上分享每天跑步的截图。
  • 用笔记本记录要做的重要之事，少用手机。

至于 Obsidian，就作为知识体系来维护吧，当做演草本未免太过沉重和臃肿。
